## 1. Product Vision

**コンセプト: 「東京の秘密の遊び場」**

東京の見どころ（浅草・新宿・渋谷）以外の地域を体験したい旅行者へ、ローカルな真実の体験を提供するプラットフォーム。

**なぜ存在するか:**
> 旅行者：「東京の見どころ以外を体験したいけど、行くのが怖い。ハズレたくない。」
> MATCHS：「怖くない。はずだ。当たりしかない。」

**、成功の定義:**
- ユーザーが迷わず最短で「当たり」を引ける
- 体験前の期待と体験後の満足が一致する（期待値管理）
- 「また来たい」と「誰かに勧めたい」が自然に発生

## 2. Target User Problems

### ユーザーの深い悩み（Priority Order）

| # | 悩み | 深さ | 競合解決度 |
|---|---|---|---|
| 1 | **ガイドを信頼できるか分からない** | 極深 | △ 名前と写真があるだけ |
| 2 | **何を予約すべきかわからない** | 深 | △ 数が多すぎて選べない |
| 3 | **失敗した時のリスクが怖い** | 深い | ○ キャンセルポリシーあるが冷たい |
| 4 | **複数体験の計画が面倒** | 普通 | ✗ そういう機能がない |
| 5 | **ディープな情報が欲しい** | 普通 | ○ ガイドブックには載っていない |

**핵심 Insights:**

- 旅行者は「失敗しない選択」を本能的に求めている
- 情報は多いが判断材料がない（評価点数だけでは足りない）
- 「ガイドがどんな人か」が見えるかが最大の信頼信号

## 3. Competitive Analysis

### 主要競合

| 競合 | 強み | 致命的な弱み |
|---|---|---|
| Airbnb Experiences | ブランド力・安心感・量が豊富 | ホストが匿名・パーソナライゼーションなし |
| GetYourGuide | 信頼性・Instant Booking対応 | 手数料高い・ホストの个性为零 |
| Voyagin/JAPANiCAN | 日本特化・安心感 | UXが古い・人格がない |

### 競合の共通弱点（ここに刺す）

1. **ホスト人格化がない** - 名前と顔写真だけ。人は物語で動く。
2. **Expert Curatedがない** - 旅行者は「best of」を求めているので、迷う時間がない。
3. ** Compatibility Matchingがない** - 「自分に向いているか」が分からない。

## 4. Solution Strategy

### 4つの柱（Priority順）

#### P1: ホスト人格化 (Host Humanization)
**目的:** ユーザーが「ceral」を感じる

```
Implementation:
- 各ガイドに「人生の物語」を300文字で記載
- 動画30秒自己紹介（任意）
- Villegeでの生活風景画像3枚
- 「前身職業」「趣味」「このエリアが好き理由」
```

#### P2: Expert Curated Lists
**目的:** 迷う時間をゼロにする

```
Implementation:
- TOPページに追加：「最初の体験に最佳3選」
- 「地元民しか知らない穴場3選」
- 「子連れOK・雨の日でも大丈夫」
- 「これがなきゃ始まらない」セクション
```

#### P3: Compatibility Matching
**目的:** 期望值を正しく管理し不満を低減

```
Implementation:
- 体験詳細ページ：「こんな人に向いてる」バッジ
  - アクティブ度: のんき / 雰囲気になる / がっつり動く
  - 一人赴き / グループ
  - 食事への興味:  массовая еда / 繊細な味覚
- >Optional: 3問クイッククイズで最适合体験推荐
```

#### P4: 安心保証の刷新 (Trust System)
**目的:** 「怖い」を消す

```
Implementation:
- 「満足保証，否则退款」+ 具体的に條件を記載
- 体験前の無料監督は60分ビデオ通話
- 体験後のフィードバックで次回割引码発行
- ガイドとの مباشر連絡（WhatsApp）許可
```

## 6. Design Principles

### 従うべきルール

1. **迷わせない** - ユーザーは1回の閲覧で判断する。情報は整理されてワケない。
2. **人格を见せる** - 每个ページ每个コンポーネントに「人」を感じる要素を入れる。
3. **怖さを消す** - 退款保証、ガイドとの面と电话、レビューは正直に。嘘は×。
4. **结果导向** - 各改善はCVR・直帰率・リピート率のどれかに帰属する。

### やってはいけないこと

- 情報が罗列出ないようにする（精选3项）
- 评价点数だけの信赖にならない（点数以下の评論を読む）
- -commercializeしすぎない（体验自体が储けの下地）

## 8. Content Guidelines

### 体験说明文の写法

```
Bad: 「The tour includes food and drinks. You will visit 3 restaurants.」
Good: 「元寿司职人の田中さんが教你吃的正确方式。去了3家，只有当地人知道的店。」
```

- ガイド persoименя的语气で書く
- 具体的な专有名词を入れる（「田中さん」「要高野哪家店」）
- 危险我的话は×（「とても良かった」→具体的な描写）

### ホスト紹介文の写法

```
必要な要素（顺不同）:
- コラー生涯の語（职业・趣味・このエリアに来た理由）
- 最近のアクティビティ（「今月は每周末、八王子でバーを经营」）
- 游客へのメッセージ（1文）
- 写真3枚（ Villege + 活动 + 笑っている）
```
